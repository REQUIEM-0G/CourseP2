# Відтворюваність ML-результатів

Для забезпечення ідентичності результатів навчання та роботи моделі в різних середовищах (Dev, CI, Prod) впроваджено наступні механізми:

## 1. Фіксація генераторів випадкових чисел (Seeds)
У коді `app.py` та скриптах навчання жорстко зафіксовано початкові значення:
```python
np.random.seed(42)
tf.random.set_seed(42)
random.seed(42)
```

## 2. Технологічний стек
*   **Python**: 3.10 (фіксовано у Docker та CI).
*   **TensorFlow**: 2.15.x (strict pinning у requirements.txt).
*   **NumPy**: 1.24.x (запобігання конфліктам типів DType).

## 3. Артефакт моделі
*   Файл `mnist_model.h5` включений до репозиторію як незмінний артефакт.
*   Валідація цілісності моделі виконується скриптом `check_model.py` при кожному коміті.

## 4. Пайплайн передобробки
Інференс використовує ідентичний алгоритм обробки зображень, що і під час навчання:
1. Grayscale conversion.
2. OpenCV `INTER_AREA` interpolation для ресайзингу.
3. Нормалізація значень пікселів у діапазон [0, 1].
