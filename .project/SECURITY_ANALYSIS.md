# Аналіз безпеки (DevSecOps)

У проєкті реалізовано принцип «Security as Code». Безпека інтегрована в CI/CD пайплайн.

## 1. Захист від атак на рівні вводу
*   **Image Bomb Protection**: Система примусово виконує ресайзинг будь-якого вхідного зображення до 28x28 пікселів за допомогою OpenCV до того, як дані потраплять у пам'ять моделі.
*   **Типізація**: Перевірка вхідних даних на відповідність типу `float32` та розмірності тензора (1, 28, 28, 1).

## 2. Безпека ланцюжка постачання
*   **pip-audit**: CI-пайплайн автоматично сканує `requirements.txt` на наявність відомих вразливостей (CVE) у PyPI Advisory Database.
*   **GitHub Dependabot**: Налаштовано щотижневе сканування та автоматичне створення Pull Requests для оновлення застарілих бібліотек.

## 3. Безпека контейнера
*   **Slim Images**: Використання образу `python:3.10-slim` мінімізує поверхню атаки за рахунок відсутності зайвих утиліт (SSH, компілятори).
*   **User Isolation**: Застосунок працює в ізольованому просторі імен Docker, що обмежує доступ до хостової файлової системи.

## 4. Управління секретами
*   Використання `.gitignore` для запобігання потраплянню `.env` та локальних конфігурацій у публічний репозиторій.
